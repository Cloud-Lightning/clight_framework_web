嵌入式框架 / 科技蓝白 / 单屏展示

看起来很远，
但结构非常清晰。

这个首页把嵌入式框架的分层逻辑做成了一个单屏可读的视觉界面：蓝白色的未来感、黑色文字的清晰度、 立体阴影带来的 3D 厚度，都服务于“统一 API、分层实现、平台复用”这件事本身。

Embedded Ready Visual System
核心板块
API

统一调用

业务层优先面对稳定 API，而不是直接耦合某个平台的 HAL 或 SDK。

Port

平台适配

把 HPM / STM32 / ESP32 差异收口到实现层，而不是把差异扩散到业务代码里。

Manifest

映射清单

用生成结果索引把 profile、实例名和底层 BSP 宏的关系讲清楚。

框架架构图
Layer 01 bsp/api

给业务代码直接使用的统一公共接口。

→
Layer 02 bsp/interface

稳定的底层统一接口边界。

→
Layer 03 bsp/port

真实平台行为在这里完成适配。

→
Layer 04 SDK / HAL

最终落到芯片厂商或平台底层驱动。

视觉方向

蓝色、白色、黑字，卡片之间要有明显的空间和重量。

这次不是做成长滚动页面，而是把信息压缩在一个屏幕里。每个板块清楚分区，字号收小， 同时加入真实的 hover 倾斜感和轻量入场动画，让界面更像“嵌入式框架官网首屏”。

3D Hover

卡片 hover 时会倾斜并增强阴影，形成真正的立体感。

Single Screen

内容压缩到首屏，不再依赖长滚动叙事。

Structured Blocks

Hero、特性、架构图、视觉说明四个板块清晰分开。

Embedded Tone

整体更像技术系统首页，而不是泛化的概念海报。
